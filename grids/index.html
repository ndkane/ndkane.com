Grids, Resetting the CSS, and a Few Escaped Entities…

Escaping Entities

In the previous posts, I had come up with a small issue concerning the presentation of elements without actioning them. This led to many annoying mistakess when I tried to demonstrate what I had used in the stylesheet or HTML.

Therefore I learnt about escaping the code with HTML entities. I’d already used one without knowing it when I first attempted footnotes. I used the character entity ‘&sup1;’ before I knew how to change it in the stylesheet. Knowing about character entities, which escape the code, allowed me to fix it. Problem solved. We found a handy site by w3.org which has a big database of character entities, which I’ve found very helpful since.

Resetting the CSS

From the beginning of this project, we had been using a style sheet that was essentially working around a set of defaults. I didn’t have the freedom that I’d like, however Aral showed that there is a way to maintain complete control of my code. In order to dothis, we needed to reset the CSS.

Essentially, the goal of a reset stylesheet is to reduce browser inconsistencies. These can be anything from default base line heights, to margins and headings. Therefore we used Eric Meyer’s CSS Reset Version 2.0 and placed it within the <head> before any of the other stylesheets present, as this would be the first stylesheet the code would apply. However, our original stylesheet was still in use, so we muted it by placing it into a comment.

This reset made my website a shadow of its former self. However, it allowed to to do everything from scratch, rather than have to make things work for me. As an artist, I create things from base elements, not just by hacking at someone else’s work in order to fit around their restrictions. Therefore we created the stylesheet "styles—vertical—rhythms.css" where I would place the specific styles I had created in the original CSS. I could then go on to create my grids, and create instructions as I go along. This was placed in the <head> after the CSS reset, as any stylesheets placed later in the order override any before them.

For a brief history of CSS Resets that I found rather interesting, take a glance at Michael Tuck’s post on Six Revisions.

A Brief Intermission

Before I go any further and explain what is meant by a grid, I’d like to draw your attention to this monstrosity:

h1 36px
h2 24px
h3 16px

I attempted to teach myself how to create a table so that I could demonstrate vertical rhythm, however, as you can see, it didn’t quite work. I wanted to create a nicely spaced table with background colour that also had spaces between the cells. I have tried the following:

  • Realising that because I reset the CSS the default settings won’t apply, therefore explaining why it isn’t quite formatting properly
  • Adding padding and margins into the table element, however that didn’t work for reasons I don’t quite understand.
  • Asking Stackoverflow and the Internets, but I don’t think I’m asking the right question perhaps.

After asking Aral, I realised that I was looking at the wrong elements altogether. Rather than trying to change the properties of the <table> element, I should have been looking at the elements that control the rows and cells of a table — the <td> and <tr> tags. It was there that I sorted out the margins and padding, which was the problem all along. There’s still an issue with the top margin/padding, but I reckon that might have to do with the padding on <p>!

Grids

As mentioned earlier, I had reset the CSS so that I could work from a blank canvas. This meant that I had to define a few values in terms of headings and line height. This was relatively simple. I just had to define a base line height, which was set at 24px, and then make all values divisible by 24. This lets the page read better. I defined the line height in the CSS under ‘body’, and then went to work ensuring my margins, padding and so on were divisible. Aral helped me to check that it fit by using a 24px grid background image, which I can easily mute when I don’t need it.

As an example, this is the grid I used for my headings. With the font size set at 16px, this grid allows for proportional distinction from the paragraph text. As you might notice, h3 is set at 16px, however, I’ll be setting it to bold instead of changing the size.

h1 36px
h2 24px
h3 16px

This makes me happy. I like that I don’t have to guess margins and paddings to see if they work, I have a basic grid that helps me to determine them. Secretly I’m happy that the line height is also the day of the month I was born on. The month itself is divisible by it too.
