Fixing last week’s problems…

Last week I mentioned problems with the margins between my list items in unordered and ordered lists. I wanted to change them as any list item that were longer than one line just looked cluttered. I’d been making the mistake of adding the margins onto the lists themselves, rather than the items. This was easily fixed by tweaking the <li> element. I’ve been learning to use Safari’s Web Inspector more and more, rather than just guessing!

I also wanted to solve the padding on tables, as before nothing that I applied in the CSS would change the top padding. This I taked through with Aral, after inspecting the elements. After a bit of experimentation we found that anything I applied to <tr> would have no effect. Only padding on <td> would apply! Problem solved. While we were looking at tables, we came across pseudo-classes.

Pseudo-Classes, or How to Organise Your Children

My tables needed a bit of jazzing up. Keeping them in a block light grey didn’t really look very nice, and weren’t very clear. Therefore I needed to look at pseudo-classes so that I could apply alternating colour in the cells. Pseudo-classes, in simple terms, is applying a selective rule to the application of an element.

Here’s the code as it appears in my stylesheet:

tr
{
  background-color: #CDCDCD;
}

tr:nth-child(2n)
{
  background-color: darkgrey;
}

tr:not(:last-child)
{
  border-bottom: 1px solid darkgrey;
}

Here you can see that we set the table rows to be #cdcdcd, and then set a pseudo-class later in the order for every other (starting at the second) cell to be a darker grey. You can also set it differently, so that rather than setting a ‘base’ color and then using a pseudo-class to override it, the initial selector would be a pseudo-class, instead defining the nth child as 0n, from the first cell, then every other. However, it makes sense to do it the way that we’ve set it, as it’s simple to order in the CSS.

I’ve also shown the selector for the row border, which uses the not: selector, indicating that the border should apply to every cell but the bottom table row. I’ll be playing with these as we go along, applying them to other selectors now that I’ve understood the format.

Navigation

Everyone needs a bit of direction, so I decided it was time to sort out my navigation. The website you see before you is the most up-to-date version, and therefore you’ll notice a few things that we fixed in later lessons. However, one of the main problems with navigation was the reponsiveness of my website after adding it. I aligned my navigation to fit the max-width (in pixels) of my page by using the first-child pseudo-class, keeping it to the right. This meant that only the first item would have a larger margin, pushing with it the others. However, this broke my site, and made it look worse than it does on mobile! We’re going to fix this next week, as I don’t want to embarrass myself in front of all my new RWD friends.

A note on Rubber Ducks, and how to find solutions

Aral has instilled within me the motto ‘Think about the solution first, then find the code’. As coding is pretty much a completely new thing to me, I’m not always going to instinctively know the answer. However, I will be able to demonstrate how I’d like to get there, or what I’d like to do. Also, I’ve learnt there’s no harm in trying, or failing, because it just means you can try something else.

One tip that friend and super-technologist Natalia Buckley passed on, was that of Rubber Ducking. It’s a good way of finding solutions by making yourself think out loud. Essentially you have a Rubber Duck, or whatever small mascot you can find, and you talk your problems through with them. I don’t mean problems like what to have for lunch, rather for asking questions like ‘Why won’t my navigation headers stay where I want them to?’. Then you explain what you’ve tried, and where you could go next, then try it, and ask again if it fails. Natalia mentioned that often you’ll unconsciously find the solution because you hadn’t considered something. This is working for me, however at the moment I don’t have a rubber duck, just a rather bemused pair of cats.
